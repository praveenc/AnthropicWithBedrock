Amazon Titan Image Generator model

Titan Image Generator G1 is an image generation model. It generates images from text, and allows users to upload and edit an existing image. Users can edit an image with a text prompt (without a mask) or parts of an image with an image mask. You can extend the boundaries of an image with outpainting, and fill in an image with inpainting. It can also generate variations of an image based on an optional text prompt. Titan Image Generator includes watermarking on the output files.

For more information on Titan Image Generator prompt engineering guidelines, see Amazon Titan Image Generator Prompt Engineering Best Practices.

Features

  • Text-to-image (T2I) generation â Input a text prompt and generate a new image as output. The generated image captures the concepts described by the text prompt.

  • Finetuning of a T2I model â Import several images to capture your own style and personalization and then fine tune the core T2I model. The fine-tuned model generates images that follow the style and personalization of a specific user.

  • Image editing options â includes inpainting, outpainting, generating variations, and automatic editing without an image mask.

  • Inpainting â Uses an image and a segmentation mask as input (either from the user or estimated by the model) and reconstructs the region within the mask. Use inpainting to remove masked elements and replace them with background pixels.

  • Outpainting â Uses an image and a segmentation mask as input (either from the user or estimated by the model) and generates new pixels that seamlessly extend the region. Use precise outpainting to preserve the pixels of the masked image when extending the image to the boundaries. Use default outpainting to extend the pixels of the masked image to the image boundaries based on segmentation settings.

  • Image variation â Uses an image and an optional prompt as input. It generates a new image that preserves the content of the input image, but variates its style and background.

  • Training samples: min: 5, max: 10,000

  • Prompts length: min: 3 char, max: 1,024 char

Parameters

  • Mask prompt â Enter a prompt to select the segmentation of the mask image. A more precise mask prompt results in an improved image mask.

  • Negative prompt â Items or concepts that you don't want the model to generate, such as cartoons or violence.

  • Reference image â The image on which to generate the response or that you want the model to edit. Size limit is 1280 x 1280 pixels.

  • Response image â Output settings for the generated image, such as quality, orientation, size, and the number of images to generate.

  • Advanced configurations â The inference parameters to pass to the model including prompt strength and seed values. Use the same values to reproduce a variation.

Training dataset for finetuning

For the training dataset, create a .jsonl file with multiple JSON lines. Each JSON line contains both an image-ref and caption attributes similar to Sagemaker Augmented Manifest format. Auto-captioning is not currently supported.


   {"image-ref": "s3://bucket-1/folder1/0001.png", "caption": "some text"}
   {"image-ref": "s3://bucket-1/folder2/0002.png", "caption": "some text"}
   {"image-ref": "s3://bucket-1/folder1/0003.png", "caption": "some text"}

The Amazon S3 paths need to be in the same folders where you have provided permissions for Amazon Bedrock to access the data by attaching an IAM policy to your Amazon Bedrock service role. For more information on granting an IAM policies for training data, see Grant custom jobs access to your training data.

Output

This model adds an invisible watermark to all generated images to reduce the spread of misinformation. In the future, a watermark detection API will be available to help you confirm whether an image was generated using the Titan model, by checking for the existence of this watermark.

Prompt Engineering Guidelines

Mask prompt â This algorithm classifies pixels into concepts. The user can give a text prompt that will be used to classify the areas of the image to mask, based on the interpretation of the mask prompt. The prompt option can interpret more complex prompts, and encode the mask into the segmentation algorithm.

Image mask â You can also use an image mask to set the mask values. The image mask can be combined with prompt input for the mask to improve accuracy. The image mask file must conform to the following parameters:

  • Mask image values must be 0 (black) or 255 (white) for the mask image. The image mask area with the value of 0 will be regenerated with the image from the user prompt and/or input image.

  • The maskImage field must be a base64 encoded image string.

  • Mask image must have the same dimensions as the input image (same height and width).

  • Only PNG or JPG files can be used for the input image and the mask image.

  • Mask image must only use black and white pixels values.

  • Mask image can only use the RGB channels (alpha channel not supported).
