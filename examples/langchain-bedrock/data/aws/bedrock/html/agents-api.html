  1. To set up an agent, you call the following Agents for Amazon Bedrock APIs.

    1. CreateAgent â Create and configure an agent and define what the agent should be able to do for customers.

    2. CreateActionGroup â Create action groups to define API calls that the agent should make to carry out the tasks it is defined to carry out.

    3. (Optional) AssociateKnowledgeBase â Associate knowledge bases with the agent so that it can query it in orchestration.

  2. To test your agent, send an InvokeAgent request through a runtime endpoint (see Agents for Amazon Bedrock RunTime APIs).

  3. Use update APIs (UpdateAgent and UpdateAgentActionGroup) to iterate on your agent.

  4. To deploy your agent, send a CreateAgentAlias request to create a version and alias and to associate the alias to the version. Set up your application to make InvokeAgent requests to this alias.
